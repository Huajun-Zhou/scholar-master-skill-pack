# Thinking Model: 图即安全透镜

## 1. 现象/缺口

传统代码/应用安全分析依赖特征工程和规则匹配，但攻击者可通过混淆、加密、Native代码等技术隐藏恶意特征。文本/序列表示难以捕捉代码的结构语义。

## 2. 问题重构

从"提取恶意代码特征"重构为"将代码/应用/系统转化为图结构（AST+控制流+数据流），用GNN在图拓扑上学习区分恶意和良性模式——图结构对混淆天然更鲁棒"。

## 3. 关键假设

- 恶意行为在图结构上留下可区分的拓扑模式
- 图表示比文本/序列表示更本质地刻画代码语义
- 攻击者改变图结构的代价高于改变文本特征

## 4. 方法机制

1. **图构建**：源代码→AST/CFG/DFG→多模态图（节点=语义单元,边=依赖关系）
2. **图学习**：GNN消息传递捕捉局部和全局图结构模式
3. **攻防验证**：从攻击者角度测试图检测的盲点，反馈改进
4. **跨域迁移**：将图安全分析范式迁移到新领域

## 5. 实验验证

- 多种图表示（AST only/CFG only/联合）的性能对比
- 代码混淆前后的检测率变化
- 跨语言/跨平台的泛化测试

## 6. 不适用场景

- 无结构信息的纯数据流（文本流、数值流）
- 图过于庞大导致GNN不可扩展
- 攻击者可以完全控制图结构

## 7. 迁移模板

问自己：
1. 我的安全问题中，对象有什么天然的结构关系？
2. 这种结构关系能否表示为图（节点+边）？
3. 恶意和良性在图拓扑上如何区分？
4. GNN能否从图结构中学习到区分性模式？
5. 攻击者改变图结构的代价有多大？
