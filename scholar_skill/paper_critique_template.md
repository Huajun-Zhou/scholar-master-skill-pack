# Paper Critique Template — 论文审查模板

按陈志远教授公开成果中体现的证据标准与研究范式审查论文。
**以下审查维度基于其 多篇论文中的稳定模式 (B 类综合)，应用于新论文时属于 C 类迁移。**

## 审查维度

### 1. 问题重要性 (Is the problem important?)
- [ ] 问题是真实存在的还是人为构造的？
- [ ] 问题的学术抽象是否合理？
- [ ] 是否清晰说明了问题为什么重要？
- 参考标准: 该学者偏好"具有明确可建模前向过程的问题"和"可挑战隐含假设的问题"

### 2. 问题定义质量 (How well is the problem defined?)
- [ ] 问题是否被形式化为可建模问题？
- [ ] 关键假设是否明确列出？
- [ ] 假设是否合理？有无隐含假设未声明？
- 参考标准: 该学者明确区分"论文说明的假设"与"隐含假设"

### 3. 方法必要性 (Is the method necessary?)
- [ ] 为什么现有方法不够？是否清晰论证了现有方法的隐含假设不成立？
- [ ] 该方法的核心 insight 是什么？是在哪个层面创新（问题抽象/求解机制/架构设计）？
- [ ] 是否只是模块的堆叠？
- 参考标准: 该学者偏好在"问题抽象层"创新，而非堆叠模块

### 4. 方法机制清晰度 (Is the mechanism clear?)
- [ ] 方法的核心机制是否用机制语言解释清楚（不只是罗列公式）？
- [ ] 能否回答"为什么有效"？
- [ ] 输入-输出-条件是否明确？
- 参考标准: Method Card 的 Section 4 标准

### 5. 证据充分性 (Is the evidence sufficient?)
- [ ] 理论推导是否完整？
- [ ] 合成实验是否覆盖可控条件？
- [ ] 真实数据验证是否充分（≥2数据集，≥5 baseline）？
- [ ] 消融研究是否深入（至少 3 层）？
- [ ] 是否包含鲁棒性分析和失败案例分析？
- [ ] 是否报告了统计显著性？
- 参考标准: Phase 4 evidence_standards.md

### 6. 实验设计合理性 (Is the experimental design sound?)
- [ ] Baseline 选择是否有代表性（经典+最新）？
- [ ] 评价指标是否合理？
- [ ] 消融设计是否真正测试了核心 claim？
- [ ] 是否区分了必要性消融和设计选择消融？
- 参考标准: 该学者的 4 层消融模式

### 7. 贡献真实性 (Are the contributions real?)
- [ ] 性能提升是 marginal 还是 significant？
- [ ] 贡献是否被过度声明？
- [ ] "first" claim 是否准确？
- 参考标准: 该学者的诚实贡献表述习惯

### 8. 局限性诚实度 (Are limitations honestly discussed?)
- [ ] 是否明确列出了方法局限？
- [ ] 是否讨论了不适用场景？
- [ ] 是否诚实说明了方法的边界条件？
- 参考标准: 该学者明确区分"论文自述局限"与"可观察局限"

## 审查报告模板

```markdown
# 论文审查报告

## 总体评分
[1-10] — [一句话总结]

## 逐项评审
### 1. 问题重要性 [评分/10]
[评价 + 建议]

### 2. 问题定义质量 [评分/10]
[评价 + 建议]

### 3. 方法必要性 [评分/10]
[评价 + 建议]

### 4. 方法机制清晰度 [评分/10]
[评价 + 建议]

### 5. 证据充分性 [评分/10]
[评价 + 建议]

### 6. 实验设计合理性 [评分/10]
[评价 + 建议]

### 7. 贡献真实性 [评分/10]
[评价 + 建议]

### 8. 局限性诚实度 [评分/10]
[评价 + 建议]

## 关键问题
1. [最需要解决的核心问题]
2. [次要问题]

## 改进建议
1. [具体、可操作的建议 — 引用陈志远教授方法论作为参考]
2. [如适用]

## 参考方法论来源
- [paper_id]: [具体方法论点]
```

**注意**: 审查中的"参考标准"基于陈志远教授公开论文的 B 类综合归纳。审查结论是对被审论文的评价，不表示陈志远教授本人持该观点。
